Proof of concept — functional but untested

Collate is a working proof of concept built by a solo developer who is a practising litigation lawyer, not a software engineer. The entire application — Rust parsing engine, React interface, test suite — was built in a single session using AI-assisted development with Cursor and Claude. It is functional and includes 113 automated tests, but it has not been tested by users in practice and has not undergone independent code review or security audit. Use it to explore the concept. Do not rely on it for live matters without testing it yourself first on documents you are comfortable with.

A checklist, not an autopilot

Collate is deliberately not a redlining tool. It does not produce a merged document. It does not auto-accept changes. It does not generate a new .docx with everyone's edits baked in.

Automated document merging is fragile. Word's own Combine function routinely corrupts formatting and misaligns changes. In litigation, where a misplaced paragraph number or a broken cross-reference can undermine a filing, that risk is unacceptable.

Collate gives you a complete, structured view of every piece of feedback, organised by paragraph, with a resolution checklist. You then open your master draft in Word and make the edits yourself — with the confidence that you have seen every item and recorded your decision on each.

You control the master

Every edit in the final document is one you chose to make, in Word, with full formatting control.

Nothing is missed

The checklist ensures every comment, every change, every phone note is explicitly addressed.

Defensible process

Export a complete record of what you accepted, rejected, and why. An audit trail for the file.

Human judgment

You decide which reviewer to follow when they disagree. No algorithm makes that call.

Coming next: structured AI analysis

Planned feature — not yet implemented

A future update will add optional AI analysis of the collated output:

  • Flag conflicts between reviewers — incompatible changes, contradictory comments
  • Identify cross-document inconsistencies — does accepting a change here contradict something elsewhere?
  • Highlight substantive issues — missing cross-references, undefined terms, unsupported claims
  • Suggest drafting improvements — document-type-aware, not generic text generation

AI analysis will be advisory, not executive. Suggestions appear alongside the checklist for you to consider. You review each one, accept or dismiss it, and make the edit yourself. The final document is always yours.

Privacy

Collate processes your documents entirely on your machine. No file is uploaded to any server. No data is transmitted anywhere. The parsing engine runs locally in WebAssembly.

A Content Security Policy header structurally prevents the browser from making outbound requests. This is not a promise — it is an architectural constraint enforced by the browser itself. Disconnect from the internet and the tool still works.

Limitations

  • Large documents (500+ pages) may be slow to parse
  • Complex nested track changes may not render perfectly
  • Only .docx format (not .doc or PDF)
  • Approximate paragraph matching across reviewer versions
  • Single-user tool — no real-time collaboration
  • Does not produce a merged output document — you edit the master in Word

How this was built

Collate was built in a single session of approximately 90 minutes by a practising UK litigation lawyer with no formal software engineering training, using AI-assisted development with Cursor and Claude. The Rust/WASM parsing engine, React frontend, 113-test suite, and this landing page were all produced in that session. The source code is public and the tool is free.

This is a demonstration of what a domain expert with access to AI coding tools can build — a working, testable application that would have taken a traditional development team three to four months and cost six figures. The person who understands the problem is the person who builds the solution. No requirements lost in translation. No meetings.
